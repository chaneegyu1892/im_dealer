- 같은 파일 `:70-71`: `bg-primary/10` → `bg-brand/10`, `text-primary` → `text-brand` (레거시 navy 대신 브랜드 색)

### 커밋 2 — 공개 화면 UX 수정 (B그룹)

**B1. 견적 페이지 모바일 헤더 겹침 해결**
- 원인: `(public)/layout.tsx`가 `/quote`에서도 `<Header/>`(z-50)를 렌더하고, 견적 미니 헤더(z-40)가 그 아래 깔림
- 해결: `src/app/(public)/quote/page.tsx`(또는 QuoteClientPageV2 최상단)에서 pathname 기반으로 Header 렌더를 우회. 가장 안정적인 방법은 `(public)/layout.tsx`에서 `usePathname`으로 `/quote` 시작 경로일 때 Header/BottomNav를 숨기는 것. 그러나 layout.tsx는 서버 컴포넌트이므로, 작은 클라이언트 래퍼를 두거나, 견적 미니 헤더의 `z-index`를 50 이상으로 올리고 사이트 헤더를 겹치지 않게 처리. **실제 구현 시 더 안전한 경로 우선.**

**B2. 견적 STEP 2 트림 fetch 데드엔드 해결**
- `QuoteClientPageV2.tsx:228-291`: `.catch(() => {})`에 에러 state(`trimsError`) 설정 추가
- `Step2ConditionV2.tsx`: `trimsError`일 때 에러 메시지 + 재시도 버튼 노출 (기존 `canRequestConsultation` 로직 유지)

**B3. 모바일 뒤로가기 폴백**
- `QuoteClientPageV2.tsx:773`: `router.back()` → 히스토리 없을 때 `/cars/${slug}`(또는 `/cars`)로 폴백
- `window.history.length <= 1` 체크 후 폴백

### 주의사항 / 리스크
- **AGENTS.md 준수**: quote-calculator.ts(금융 명세)는 건드리지 않음. admin 색상은 명세 hex 유지.
- **C1 데드 코드 삭제**: 참조 0건을 grep으로 확정했으나, 동적 import나 문자열 참조가 있는지 삭제 전 한 번 더 재확인.
- **B1 헤더 우회**: layout.tsx가 서버 컴포넌트라 `usePathname`을 직접 못 씀. 작은 클라이언트 래퍼 컴포넌트 추가가 필요할 수 있음 — 구현 시 최소 변경으로 처리.
- 변경량이 많아 4개 커밋으로 분리, 각 커밋 후 typecheck/lint 통과를 전제로 진행.
