## 3. AI 추천 로직 개선 🟡

### 현재 상태 (4단계 구조)

- **메인 컴포넌트:** `src/components/recommend/RecommendFlow.tsx`
- **옵션 데이터:** `src/constants/recommend-options.ts`
- **단계별 컴포넌트:**
  - `StepIndustry.tsx` — 1단계 업종
  - `StepPurpose.tsx` — 2단계 목적
  - `StepBudget.tsx` — 3단계 예산 (납입 성향이 같이 들어가 있음)
  - `StepPreference.tsx` — 4단계 (주행거리 + 계약 종료 방식 + 연료가 같이 들어가 있음)

### 변경 명세 — **요청은 6단계 구조**

> ⚠ 현재 4단계 컴포넌트를 6단계로 재구성해야 함. 단순 옵션 교체가 아니라 **컴포넌트 분리** 필요.

#### 3.1 1단계 업종 (4개 → 3개) 🟢

- **파일:** `recommend-options.ts:3-8`, `StepIndustry.tsx`
- 변경: `법인 / 개인사업자 / 직장인 / 개인(비사업자)` → `법인 / 개인사업자 / 개인(직장인+프리랜서+비사업자 통합)`
- 영향: 2단계 목적 매핑 테이블 재정의 필요 (`recommend-options.ts:25-42`)

#### 3.2 업종별 추가 질문 🟡

- **법인:** "현재 운용 대수" (없음 / 1대 / 2대 이상)
- **개인사업자:** "사업자 명의 렌트/리스 운용 여부" (없음 / 1대 / 2대 이상) + 안내: "2대 이상 시 임직원 전용보험 필수"
  - 🟡 **팩트체크 필요:** 구매차 포함 여부
- **개인:** "탑승 인원" (혼자 / 2~3명 / 4명 이상)
- **삭제:** 기존 "4대 보험 가입 여부", "대중교통 이용 빈도" → 코드에 미구현 상태이므로 단순 추가만 진행

#### 3.3 2단계 목적 🟢 + 🟡 보류

- 업종별 목적 재정의:
  - **법인:** 출퇴근/업무용(통합), 화물/배달, 임원용/의전
  - **개인사업자:** 출퇴근/업무용, 화물/배달, 가정용
  - **개인:** 출퇴근/업무용, 화물/배달, 가정용
- 🟡 **보류:**
  - 2단계 주행거리 중복 질문 재검토
  - 목적별 세분화 추가 질문 — **1주일 추가 논의** 후 반영

#### 3.4 3단계 예산 (6단계 → 3단계) 🟢

- **파일:** `recommend-options.ts:50-56`
- 변경: `30 이하/30~50/50~70/70~100/100~150/150 이상` → `50만원 이하 / 50~100 / 100 이상`

#### 3.5 4단계 납입 성향 (워딩 개선 + 분리) 🟢

- **파일:** `recommend-options.ts:58-78`, 현재 `StepBudget.tsx` 안에 포함됨 → **독립 단계로 분리** 필요
- 라벨 변경:
  - `보수형` → "초기 비용 없이 가볍게 시작하고 싶어요"
  - `표준형/공격형` 통합 → "보증금·선납금을 넣고 월 비용을 낮추고 싶어요"
- 삭제: "예산을 꼭 지켜야 해요" 질문 (이미 없으면 패스)

#### 3.6 5단계 주행거리 (4개 → 3개) 🟢

- **파일:** `recommend-options.ts:80-85`, 현재 `StepPreference.tsx`에 포함 → **독립 단계로 분리** 필요
- 변경: `2만 이하/2~3만/3~4만/4만 이상` → `1만/2만/3만`
- 라벨:
  - "정말 많이 타는 편이에요 (연 3만km)"
  - "적당히 타요 (연 2만km) ← 80% 고객 선택" (기본값)
  - "적게 타요 (연 1만km)"
- 평균 주행거리 통계 안내 박스 추가

#### 3.7 6단계 연료 방식 🟢

- **파일:** `recommend-options.ts:188-193`
- 라벨 정리: `전기차 / 하이브리드 / 내연기관 / 상관없음`
- 선호 시 가점, 조건 불일치 시 안내 메시지 추가 (예: "전기차는 예산이 더 필요해요")

#### 3.8 계약 종료 방식 삭제 🟢

- **파일:** `recommend-options.ts:195-214`, `StepPreference.tsx:66-88`
- 삭제: 반납형/인수형/모르겠음 질문 전체
- 영향 확인: 추천 점수 계산 로직(`src/lib/recommend-*.ts`)에서 `endType` 참조 여부 확인 후 제거
