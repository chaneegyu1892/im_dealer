## 전체 계산 파이프라인

아래 7단계는 순서가 고정된 누적 구조다. 각 단계는 이전 단계 결과에 곱하는 방식이며, **독립 합산 방식이 아니다**.

```
[Step 1] 기준 대여료      = 차량가격 × 선형보간_회수율(기간, 주행거리)
[Step 2] 조건 대여료      = 보증금 또는 선납금 적용 (둘 중 하나만, 동시 적용 불가)
[Step 3] 금융사별 조건_대여료 오름차순 정렬 → 순위 부여
[Step 4] 순위가산_대여료  = 조건_대여료 × (1 + 순위가산율)
[Step 5] 차량가산_대여료  = 순위가산_대여료 × (1 + 차량가산율)
[Step 6] 최종_대여료      = 차량가산_대여료 × (1 + 금융사가산율)  ← 여기서만 Math.round
[Step 7] 최종_대여료 기준 재정렬 → 순위 재할당
```

**중간 단계(Step 1~5)는 float을 유지. Math.round는 Step 6에서만.**

## 선형보간 공식

```
rate(x) = rate_min + (rate_max - rate_min) × (x - price_min) / (price_max - price_min)
```

- `x < price_min` → `rate_min` 사용 (클램핑)
- `x > price_max` → `rate_max` 사용 (클램핑)
- 외삽(extrapolation) 불허

```typescript
const t = Math.max(0, Math.min(1, (x - x0) / (x1 - x0)));
return y0 + t * (y1 - y0);
```

## 보증금 적용 공식

```
보증금_할인율 = (보증금비율 / 10) × 0.000523  ← depositDiscountRate = -0.000523 (음수 저장)
적용_회수율  = 기준_회수율 + depositDiscountRate × (보증금비율 / 10)
조건_대여료  = 차량가격 × 적용_회수율
```

### 검증값 (오릭스, 43,840,000원, 2만km, 36개월, 보증금 10%)

| 항목 | 값 |
|------|----|
| 기준 회수율 | 1.3337% (0.013337) |
| 보증금 할인율 | 0.0523% (0.000523) |
| 적용 회수율 | 1.2814% (0.012814) |
| **조건 대여료** | **561,680원** |

## 선납금 적용 공식

```
선납_할인율  = (선납금비율 / 10) × 0.000073  ← prepayAdjustRate = +0.000073 (양수 저장)

1단계: 원금차감_대여료 = 기준_대여료 - (선납금 / 계약기간)
2단계: 조건_대여료     = 원금차감_대여료 - (차량가격 × prepayAdjustRate × 단위수)
```

> **핵심**: `prepayAdjustRate`는 **양수(+0.000073)로 DB 저장**, 계산 시 **반드시 차감(-)**

### 검증값 (오릭스, 43,840,000원, 1만km, 36개월, 선납금 10%)

| 항목 | 값 |
|------|----|
| 기준 대여료 | 540,340원 |
| 선납금 | 4,384,000원 |
| 월 원금 차감 | 121,778원 |
| 추가 할인 | 3,200원 |
| **조건 대여료** | **~421,750원** |

## 순위 가산율

| 순위 | 가산율 | 곱셈 계수 |
|------|--------|-----------|
| 1순위 | +1.0% | × 1.010 |
| 2순위 | +1.5% | × 1.015 |
| 3순위 | +2.0% | × 1.020 |
| 4순위 이상 | +2.5% | × 1.025 |

## 가산율 적용 기준

| 가산율 | 적용 대상 | 동일 적용 범위 |
|--------|-----------|----------------|
| 순위 가산율 | 조건_대여료 | 순위별 상이 |
| 차량 가산율 | 순위가산_대여료 | 모든 금융사 동일 |
| 금융사 가산율 | 차량가산_대여료 | 금융사별 상이 |

> 금융사 가산율 적용 후 순위 역전 가능 → **최종 재정렬 필수**

## 코드 구조 (`src/lib/quote-calculator.ts`)

```
lerp()                       선형보간 (클램핑 포함)

calculateMultiFinanceQuote() Steps 3~7 — 최종만 Math.round
estimateMonthly()            단일 금융사 미리보기용
calculateScenarios()         3개 시나리오 (보수/표준/공격)
```

## 자주 발생하는 실수

| 실수 유형 | 증상 | 올바른 처리 |
|-----------|------|-------------|
| 중간 Math.round | 최종값이 수십~수백원 어긋남 | Step 6에서만 반올림 |
| prepayAdjustRate 가산(+) | 선납금 할인이 안 됨, 오히려 올라감 | 반드시 차감(-) |
| 가산율 독립 합산 | 가산율이 클수록 오차 누적 | 누적 곱셈 구조 유지 |
| rankSurcharges 0부터 시작 | 1순위 가산이 없어 전체 요금 낮아짐 | 1순위=1.0%부터 |
| 외삽 허용 (클램핑 미적용) | 범위 밖 차량가에서 비현실적 요금 | t를 [0,1] 클램핑 |
