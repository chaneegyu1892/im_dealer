## 전체 계산 파이프라인

아래 5단계 구조다. 가산은 **누적 곱셈이 아니라 차량가 기준 분할 합산**이다.

```
[Step 1] 기준 대여료          = 차량가격 × 선형보간_회수율(기간, 주행거리)
[Step 2] 조건 대여료          = 보증금 또는 선납금 적용 (둘 중 하나만, 동시 적용 불가)
[Step 3] 금융사별 조건_대여료 오름차순 정렬 → 순위 부여
[Step 4] 월 추가금(i)         = (차량가격 × 가산율_i / 100) ÷ 계약개월수   (i: 순위 · 차량 · 금융사)
[Step 5] 최종_대여료          = 조건_대여료 + Σ 월 추가금     ← 여기서만 Math.round
       (이후 최종_대여료 기준 재정렬 → 순위 재할당)
```

**중간 단계(Step 1~4)는 float을 유지. Math.round는 Step 5에서만.**

## 선형보간 공식

```
rate(x) = rate_min + (rate_max - rate_min) × (x - price_min) / (price_max - price_min)
```

- `x < price_min` → `rate_min` 사용 (클램핑)
- `x > price_max` → `rate_max` 사용 (클램핑)
- 외삽(extrapolation) 불허

```typescript
const t = Math.max(0, Math.min(1, (x - x0) / (x1 - x0)));
return y0 + t * (y1 - y0);
```

## 보증금 적용 공식

```
적용_회수율 = 기준_회수율 + depositDiscountRate × (보증금비율 / 10)
조건_대여료 = 차량가격 × 적용_회수율
```

> **부호 규칙**: `depositDiscountRate` 는 **음수만 허용** (Deposit Discount 전용).
> 양수 입력은 어드민 API에서 차단된다.

### 검증값 (오릭스, 43,840,000원, 2만km, 36개월, 보증금 10%)

| 항목 | 값 |
|------|----|
| 기준 회수율 | 1.3337% (0.013337) |
| 보증금 할인율 | -0.0523% (-0.000523) |
| 적용 회수율 | 1.2814% (0.012814) |
| **조건 대여료** | **561,680원** |

## 선납금 적용 공식

```
1단계: 원금차감_대여료 = 기준_대여료 - (선납금 / 계약기간)
2단계: 조건_대여료     = 원금차감_대여료 + (차량가격 × prepayAdjustRate × 단위수)
```

> **부호 규칙**: `prepayAdjustRate` 는 **부호 자유**.
> 양수면 가산, 음수면 할인. DB 저장 부호를 그대로 더한다.

### 검증값 (오릭스, 43,840,000원, 1만km, 36개월, 선납금 10%, prepayAdjustRate = -0.000073)

| 항목 | 값 |
|------|----|
| 기준 대여료 | 540,340원 |
| 선납금 | 4,384,000원 |
| 월 원금 차감 | -121,778원 |
| 추가 조정 (음수=할인) | -3,200원 |
| **조건 대여료** | **~415,360원** |

## 가산율 적용 공식 (차량가 기준 합산)

```
월 추가금(i)   = (차량가격 × 가산율_i / 100) ÷ 계약개월수
최종_대여료    = 조건_대여료
              + 월 추가금(순위)
              + 월 추가금(차량)
              + 월 추가금(금융사)
```

| 가산율 | 적용 대상 | 동일 적용 범위 |
|--------|-----------|----------------|
| 순위 가산율 | 차량가 | 순위별 상이 |
| 차량 가산율 | 차량가 | 모든 금융사 동일 |
| 금융사 가산율 | 차량가 | 금융사별 상이 |

> 금융사 가산율 적용 후 순위 역전 가능 → **최종 재정렬 필수**

### 순위 가산율

| 순위 | 가산율 |
|------|--------|
| 1순위 | +1.0% |
| 2순위 | +1.5% |
| 3순위 | +2.0% |
| 4순위 이상 | +2.5% |

### 검증값 (고객 명세 예시)

| 항목 | 값 |
|------|----|
| 차량가 | 50,000,000원 |
| 기본 월대여료 | 500,000원 |
| 계약기간 | 48개월 |
| 가산율 (단일 1%) | 1% |
| 월 추가금 | (50,000,000 × 0.01) / 48 = 10,416.67원 |
| **최종 월대여료** | **510,417원** |

## 코드 구조 (`src/lib/quote-calculator.ts`)

```
getInterpolatedRate()        선형보간 (클램핑 포함)
calcBaseMonthly()            Step 1 — float 반환 (Math.round 없음)
applyDeposit()               Step 2 보증금 — float 반환
applyPrepay()                Step 2 선납금 — float 반환, prepayAdjustRate 부호 그대로 합산(+)
calculateMultiFinanceQuote() Steps 3~5 — 차량가 기준 합산, 최종만 Math.round
estimateMonthly()            단일 금융사 미리보기용
calculateScenarios()         3개 시나리오 (보수/표준/공격)
```

## 자주 발생하는 실수

| 실수 유형 | 증상 | 올바른 처리 |
|-----------|------|-------------|
| 중간 Math.round | 최종값이 수십~수백원 어긋남 | Step 5에서만 반올림 |
| prepayAdjustRate 차감(-) | 부호 의미가 뒤집힘 (양수가 할인이 됨) | 부호 그대로 합산(+) |
| depositDiscountRate 양수 허용 | 보증금이 가산처럼 동작 | 음수만 허용, 어드민 API에서 차단 |
| 가산율 누적 곱셈 | 가산금이 월대여료에 비례하게 됨 (정책 위반) | 차량가 × % ÷ 개월수 합산 |
| rankSurcharges 0부터 시작 | 1순위 가산이 없어 전체 요금 낮아짐 | 1순위=1.0%부터 |
| 외삽 허용 (클램핑 미적용) | 범위 밖 차량가에서 비현실적 요금 | t를 [0,1] 클램핑 |
