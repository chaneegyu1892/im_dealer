# 카카오 회원 전화번호 수집 — 설정 가이드 (팀원용)

**목적:** 카카오 로그인 회원의 **전화번호**를 수집해 관리자 **사용자 관리** 화면에 표시한다.
현재는 카카오 로그인 시 전화번호를 요청하지 않아 대부분의 회원이 "연락처 없음"으로 표시된다.

> ⚠️ **이 작업은 카카오 개발자 콘솔 + Supabase + Vercel 설정이 필요하며, 코드만으로는 동작하지 않습니다.**
> 코드는 이미 준비되어 있고(아래 "이미 되어 있는 것" 참고), **콘솔 승인 → 환경변수 ON** 순서만 진행하면 됩니다.

---

## 1. 현재 상태 / 원인

- 로그인 시 요청 scope가 `profile_nickname profile_image`뿐 → 카카오가 전화번호를 주지 않음 → 저장·표시할 값이 없음.
- 견적/본인인증을 거친 회원만 `SavedQuote.phone`으로 번호가 잡혀 일부만 보임.

## 2. 이미 되어 있는 것 (코드)

> 코드 변경은 완료되어 있으며, **환경변수 플래그가 꺼져 있어 배포해도 기존 동작과 동일**하다(안전).

| 파일 | 내용 |
|---|---|
| `src/app/(public)/login/LoginContent.tsx` | `NEXT_PUBLIC_KAKAO_REQUEST_PHONE=true`일 때만 `phone_number` scope 요청 |
| `src/app/auth/callback/route.ts` | 로그인 콜백에서 전화번호를 `User.phone`에 저장. Supabase가 안 넘겨주면 `provider_token`으로 카카오 API(`/v2/user/me`) 직접 조회해 보강 |
| `src/lib/admin-queries/users.ts` | 저장된 phone을 읽어 사용자 관리에 표시 (변경 없음, 기존 코드) |

## 3. 팀원 To-Do 체크리스트

- [ ] **A. Supabase에 연결된 카카오 앱 확인** (어떤 앱을 설정할지 특정)
- [ ] **B. 카카오 비즈니스 앱 전환**
- [ ] **C. 전화번호(phone_number) 동의항목 활성화 + 검수 신청**

- [ ] **E. 동작 확인**

---

### A. Supabase에 연결된 카카오 앱 확인 ← 먼저

카카오 로그인은 Supabase를 거치므로, **설정 대상 앱은 Supabase에 등록된 카카오 앱**이어야 한다.
(다른 앱에 설정하면 적용되지 않음 — 가장 흔한 실수)

1. **Supabase 대시보드** → Authentication → Providers → **Kakao**
2. 여기 등록된 **REST API 키(Client ID)** 를 확인
3. 카카오 개발자 콘솔에서 **그 키와 동일한 앱**을 골라 B·C를 진행

### B. 카카오 비즈니스 앱 전환

1. [카카오 개발자 콘솔](https://developers.kakao.com) → 내 애플리케이션 → 해당 앱
2. **앱 설정 → 비즈니스**
3. 사업자 정보(사업자등록번호) 등록·인증 → 비즈니스 앱으로 전환
   - 사업자 정보: 모빌페이브 (사내 확인)

### C. 전화번호 동의항목 활성화 + 검수

1. **제품 설정 → 카카오 로그인 → 동의항목**
2. **전화번호(phone_number)** → 상태를 **"선택 동의"** 로 설정 (권장; 필수로 하면 미동의 시 로그인 불가)
3. **수집·이용 목적** 입력 후 검수 신청 (영업일 며칠 소요)

**검수 목적 문구 예시 (복사해서 사용/수정):**
> 렌트·리스 견적 상담 서비스로, 회원이 요청한 견적 및 계약 진행 상황을 안내하고
> 담당 상담원이 연락하기 위해 전화번호를 수집합니다. 수집한 번호는 상담·계약 목적
> 외에는 사용하지 않습니다.

4. **Redirect URI 확인**: 제품 설정 → 카카오 로그인 → 활성화 ON, Redirect URI에 Supabase 콜백 URL이 있는지 (로그인 이미 동작 중이면 보통 OK)

> Supabase Provider 설정에도 scope가 필요하면 Supabase → Kakao provider의 scope 필드에 `phone_number`를 추가한다(콘솔 승인이 먼저).

### D. (승인 후) 활성화

1. **Vercel** → 프로젝트 → Settings → Environment Variables
2. 추가: `NEXT_PUBLIC_KAKAO_REQUEST_PHONE` = `true` (Production)
3. **재배포** (환경변수 반영을 위해)

> ⚠️ **순서 주의**: C(검수 승인)가 끝나기 **전에** 이 플래그를 켜면, 승인 안 된 scope를 요청하게 되어 **카카오 로그인이 실패**할 수 있다. 반드시 **승인 후** 켠다.

### E. 동작 확인

1. 카카오로 **새로 로그인**(또는 재로그인) — 동의창에 "전화번호 제공"이 뜨는지 확인
2. 관리자 **사용자 관리**에서 해당 회원의 "연락처 없음"이 실제 번호로 바뀌는지 확인

---

## 4. 롤백

문제가 생기면 **Vercel 환경변수 `NEXT_PUBLIC_KAKAO_REQUEST_PHONE`를 삭제(또는 false)** 후 재배포하면 즉시 기존 동작(전화번호 미요청)으로 복귀한다. 코드 되돌림 불필요.

## 5. 주의사항

- **기존 회원은 소급 수집 불가**: 이미 가입한 회원(전화번호 미보유)은 **다음 로그인 때** 재동의를 거쳐야 번호가 채워진다.
- **저장 형식**: 카카오는 `+82 10-1234-5678` 형태로 내려주며 원본 그대로 저장한다(채널톡 전달 시 `toE164KR`로 정규화).
- **카카오싱크와 무관**: 이 작업(비즈니스 앱 + 전화번호 검수)은 카카오싱크 전환과 별개로 먼저 진행 가능하다.
- **검수 지연 대비 대안**: 전화번호 검수가 지연/반려되면, 가입 직후 앱에서 전화번호를 직접 입력받는 방식(카카오 승인 불필요)으로 대체할 수 있다.
